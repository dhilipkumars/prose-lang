# PRIMES is in P — AKS Primality Test

A cross-language implementation of the **AKS (Agrawal-Kayal-Saxena) Primality Test** in Go, Rust, and Java — the first deterministic, polynomial-time, unconditional primality proving algorithm.

> Built with [prose-lang](https://github.com/dhilipkumars/prose-lang). See the [specification](src/PRIMES_is_in_P.prose) for the full design.
>
> *This document was generated by the `prose document` workflow.*

## The Algorithm

The [AKS primality test](https://en.wikipedia.org/wiki/AKS_primality_test) (2002) proved that **PRIMES ∈ P** — primality can be decided in polynomial time. The implementation follows the algorithm exactly as specified in the `@prose_strict_block`:

```
1. Check if n is a perfect power: if n = a^b for a > 1 and b > 1 → COMPOSITE
2. Find the smallest r such that ord_r(n) > (log₂ n)²
   If 1 < gcd(r, n) < n for any r checked → COMPOSITE
3. For all 2 ≤ a ≤ min(r, n−1), check if a divides n → COMPOSITE
4. If n ≤ r → PRIME
5. For a = 1 to ⌊√(φ(r)) · log₂(n)⌋:
   if (X+a)^n ≠ X^n+a (mod X^r − 1, n) → COMPOSITE
6. Output PRIME
```

Step 5 is the core — it performs **polynomial arithmetic modulo (X^r − 1, n)** using binary exponentiation, which is what makes AKS theoretically elegant but practically slow.

## Building

### Go
```bash
cd generated/PRIMES_is_in_P/go
go build -o aks_test .
```

### Rust
```bash
cd generated/PRIMES_is_in_P/rust
cargo build --release
```

### Java
```bash
cd generated/PRIMES_is_in_P/java
javac AKSPrimalityTest.java
```

## Running

Each implementation takes a single integer and prints `true` (prime) or `false` (composite):

```bash
# Go
./generated/PRIMES_is_in_P/go/aks_test 999983
# → true

# Rust
./generated/PRIMES_is_in_P/rust/target/release/primes_is_in_p 999983
# → true

# Java
java -cp generated/PRIMES_is_in_P/java AKSPrimalityTest 999983
# → true
```

### Key Takeaways

- **Go and Rust are virtually tied** at ~0.25s/number — the polynomial arithmetic dominates and both compile to comparably efficient native code.
- **Java is ~27× slower** — primarily due to JVM startup overhead per invocation (each number spawns a new `java` process). A long-running JVM application would perform much closer to Go/Rust.
- **All three languages found the exact same 100 primes** — perfect parity, confirming the deterministic algorithm produces identical results.
- **AKS is slow by design** — this is a *theoretical* breakthrough (proving PRIMES ∈ P), not a practical primality test. For production use, the Miller-Rabin probabilistic test or BPSW test are orders of magnitude faster.

- **Complexity**: O(log^(6+ε)(n)) — polynomial in the number of digits
